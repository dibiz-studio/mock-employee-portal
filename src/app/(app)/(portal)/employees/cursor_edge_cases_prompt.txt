## 2️⃣ UI / UX EDGE‑CASE AUDIT

### 2.1. Back‑Button Confusion
- **Problem:** The sidebar contains a “Back” link that calls `router.back()`. Users may think it navigates to a parent page (e.g., department list) but it actually goes back in browser history, sometimes exiting the app.
- **Fixes:**
  1. **Rename** the link to something explicit like `← Employees` or `← Departments` depending on context.
  2. **Replace** `router.back()` with a deterministic `router.push('<parent‑route>')` for each page that currently uses a generic back button.
  3. **Add** a small left‑arrow icon (from `lucide-react`) to differentiate it from a generic “Back”.
- **Files to update** (search for `router.back()`):
  - `src/features/employees/components/create-employee-form.tsx` (≈ line 226)
  - `src/features/kpi/components/create-template-form.tsx` (≈ line 180)
  - `src/features/kpi/components/assign-kpi-form.tsx` (≈ line 254)
  - Any other component using `router.back()`.

### 2.4. Edge‑Case Handling
- **Empty State** – When a list (employees, departments, KPIs) is empty, display a friendly empty‑state component with a call‑to‑action button (e.g., “Add first employee”).
- **Network Errors** – Wrap all Supabase calls in `try/catch` and funnel errors to `showErrorToast`. For 401/403 responses, redirect to `/login?error=unauthorized`.
- **Session Expiration** – Detect `supabase.auth.onAuthStateChange` events. If the session expires, automatically redirect to the login page with a toast “Session expired, please sign in again”.
- **Role‑Based UI Guard** – The `RoleGuard` component already exists. Ensure every page that requires a role wraps its content with `<RoleGuard allowed={['HR','MANAGER']}>`. Add missing guards on:
  - `settings/company/page.tsx` (SUPER_ADMIN only)
  - `kpi/templates/page.tsx` (MANAGER, HR)
  - `payroll/*` (HR, SUPER_ADMIN)
- **Form Submission Double‑Click** – Disable the submit button while the request is in flight (`isSubmitting` state). Show a spinner inside the button.
- **File Upload Limits** – If any component allows avatar upload, enforce a max size of **2 MB** and accept only `image/jpeg`, `image/png`. Show a clear error message otherwise.
- **Responsive Breakpoints** – Verify that all modal dialogs, tables, and cards collapse gracefully at `sm` (640 px). Use Tailwind’s `overflow-x-auto` on tables, and stack cards vertically on mobile.
- **Dark‑Mode Contrast** – Run Lighthouse accessibility audit in dark mode. Adjust any low‑contrast text (e.g., `text-muted-foreground` on dark background) to meet AA contrast (≥ 4.5:1).
- **Back‑Navigation Guard** – When a user is on the **Onboarding waiting page**, prevent them from navigating back to the login screen via the browser back button (use `router.replace` after login). Show a toast “Your account is pending approval”.
- **Supabase RLS** – Verify Row‑Level Security policies for `profiles`, `departments`, `kpis`:
  - **Profiles**: users can read/write only their own row.
  - **Departments**: HR/ADMIN can read all; only ADMIN can delete.
  - **KPI Templates**: only MANAGER/HR can create; only the creator can edit.
  - Add a short note in `supabase/migrations/` if policies need updating.
